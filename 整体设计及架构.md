## 我建议对规划再做一个调整（也是我认为最重要的一点）

目前规划里，**AI 模块是按"功能"划分的**（聊天、总结、出题……）。随着功能越来越多，这个模块会变得很臃肿。

我建议从一开始就按**职责**拆分：

```text
AI Service
├── ChatService        // 对话
├── PromptService      // Prompt 管理
├── EmbeddingService   // 向量生成
├── RAGService         // 检索增强
├── DocumentService    // 文档解析
└── ExamService        // 出题、批改
```

这样每个 Service 职责单一，后续你想把 DeepSeek 换成其他模型、加入重排序（Rerank）、甚至增加 Agent，都不用大改代码。

**这是我认为比较接近企业项目的架构，也是我推荐你从第一版就采用的方案。**
